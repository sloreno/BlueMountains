On what proportion of these acres will some treatment help?¶

For this the compile_Effective table will be used, which was made via the following query.

SELECT Combine_POTFIRE.biosum_cond_id, Combine_POTFIRE.rxpackage, Combine_POTFIRE.rxcycle, Combine_POTFIRE.fvs_variant, Combine_POTFIRE.Year, Combine_POTFIRE.FE_Hazard, Combine_POTFIRE.wHazard, combine_SUMMARY.Year, combine_SUMMARY.rxcycle, combine_SUMMARY.RTpa, combine_SUMMARY.netRev_FE, combine_SUMMARY.netRev_overall, cond.acres, cond.owngrpcd
FROM Combine_POTFIRE 
INNER JOIN (combine_SUMMARY INNER JOIN cond ON combine_SUMMARY.biosum_cond_id = cond.biosum_cond_id) 
ON (Combine_POTFIRE.rxcycle = combine_SUMMARY.rxcycle) 
AND (Combine_POTFIRE.rxpackage = combine_SUMMARY.rxpackage) 
AND (Combine_POTFIRE.biosum_cond_id = combine_SUMMARY.biosum_cond_id)
WHERE (((combine_SUMMARY.rxcycle)="1"));

Create a table from the cycle1_optimization table that has everything we need in this analysis.

SELECT cycle1_optimization.biosum_cond_id, cycle1_optimization.rxpackage, cycle1_optimization.rx, cycle1_optimization.change_value AS ptorchChange, tiebreaker.post_variable1_value AS TIPost, cond.acres
FROM (cond INNER JOIN tiebreaker ON cond.biosum_cond_id = tiebreaker.biosum_cond_id) INNER JOIN cycle1_optimization ON (cycle1_optimization.rxcycle = tiebreaker.rxcycle) AND (cycle1_optimization.rxpackage = tiebreaker.rxpackage) AND (tiebreaker.biosum_cond_id = cycle1_optimization.biosum_cond_id);
